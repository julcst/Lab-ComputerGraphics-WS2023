\section{Evaluation}
\label{sec:evaluation}

We implemented the ideas of \textcite{deliot2023} in GLSL using an OpenGL 4.1 forward renderer written in C++. As a base microfacet model we chose the Trowbridge-Reitz model by \textcite{trowbridge1975, walter2007} with Schlick's approximation of the Fresnel term.

\subsection{Performance}
\label{sec:performance}

\input{figures/performance.tex}

To evaluate the performance of our method we measured the render time for the same object in different resolutions using OpenGL timer queries and compared it to a standard implementation of a smooth microfacet model \parencite{trowbridge1975,walter2007}.
%
Timings were obtained on an Nvidia RTX 3060 TI for a full-screen glinty plane by varying the window resolution.
%
We observed a stable performance cost approximately $10\times$ higher than 
the smooth reference model, which makes our implementation capable of real-time (see \cref{fig:performance}).
%
As expected, the performance is roughly proportional to the total pixel area covered by the object and does not vary for different anisotropy or detail levels, as the number of binomial evaluations remains constant (see \cref{sec:discretization}).

On macOS, we experienced dramatically worse performance than on Linux and Windows, but this is likely caused by a faulty GLSL compiler implementation, that does not optimize our highly encapsulated code well, as the performance of other shaders is not affected to an equal degree.
