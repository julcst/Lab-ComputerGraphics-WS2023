\section{Evaluation}
\label{sec:evaluation}

\textbf{Implementation.} We implemented the presented layered BRDF for the isotropic and the anisotropic case in GLSL fragment shaders. Therefore, we wrote our own C++ OpenGL real-time rendering engine. Some features of our engine are:
\begin{itemize}
    \item obj model loading
    \item saving/loading scenes and material presets
    \item easy switching between shaders and models
    \item loading cubemaps
\end{itemize}
Our implementation of the layered BRDF does not rely on precomputation, instead we just use the Fresnel term as an approximation of FGD. Furthermore, we limited the number of layers to 4 (could be increased of course) and did not implement the doubling algorithm, which is not really a limitation. For each layer $\eta$, $\kappa$ and $\alpha$ can also be textured. The anisotropic shader is using a two-component vector for the variance. In our engine are currently two modes of lighting possible: a directional light or image based lighting (IBL). For IBL in the layered shaders we are using Monte Carlo integration with importance sampling as described in Section \ref{sec:anisotropic_interfaces}. The hammersley sequence is used for generating low-discrepancy points in order to sample the VNDF.

%

Each layer is parametrized by a three-component complex index of refraction $\eta + i \kappa$, a scalar/two-component roughness $\alpha$, a scalar depth $h$, a three-component absorption cross-section $\sigma_a$, a three-component scattering cross-section $\sigma_s$ and a phase function anisotropy factor $g$.\\

%

\noindent\textbf{Results.} A lot of the desired goals mentioned in Section \ref{sec:introduction} are actually fulfilled by the presented model and our implementation of it:
\begin{itemize}
    \item the model is physically based because of using the microfacet model following the GGX distribution and because it tries to approximate the physical light transport
    \item it supports an arbitrary number of layers
    \item multiple scattering is accounted
    \item if incorporated in FGD, the model is energy conserving
    \item texturing and importance sampling is supported
    \item there is no or only little precomputation and storage needed
    \item the model can be evaluated in real-time
\end{itemize}

However, there are some limitations of the general accuracy and regarding the effects the model is able to reproduce. Still, the model is able to convincingly reproduce a variety of complex layered materials. For instance metal foil, metallic paint, frosted metal, clear coating and gold coating, see Figure \ref{fig:possible_materials}.

\textit{Performance.} All results are computed on an 8 core Intel i7-1165G7 with 16 GB of RAM and an integrated Mesa Intel Xe Graphics GPU measured using OpenGL timer queries. Render times compared to the standard GGX shader are reported in Figure \ref{fig:performance_ggx_comparison}. It can be seen, that our layered shader is on average approximately 2.9 times slower, however it converges to approximately 10ms per frame for $1920\times1080$, thus it is fast enough for real-time usage.

\begin{figure}[!h]
    \includesvg[width = \columnwidth, keepaspectratio]{performance_plot.svg}
    \caption{Render times of a full-screen plane with our implementation of the layered shader with two layers compared to the standard GGX shader. Evaluated for different window sizes.}
    \label{fig:performance_ggx_comparison}
\end{figure}

Figure \ref{fig:performance_layers} shows, as expected, that the runtime is linear in the number of layers, as during evaluating the adding-doubling algorithm we iterate over all layers. Our implementation was written with readability in mind instead of performance optimization. Therefore, the render times could probably be decreased.

\begin{figure}[!h]
    \includesvg[width = \columnwidth, keepaspectratio]{performance_layer_plot.svg}
    \caption{Render times of a full-screen plane with our implementation of the layered shader for different numbers of layers at $1920\times1007$ window size.}
    \label{fig:performance_layers}
\end{figure}

In Figure \ref{fig:performance_ibl} a comparison of sample counts and render times is displayed for image based lighting using Monte Carlo integration of the anisotropic layered BRDF. This shows the limited usability of the anisotropic shader for real-time applications, as the needed sample counts for visually pleasing results are too high. The resulting images can be seen in Appendix \ref{sec:appendix_b}.
